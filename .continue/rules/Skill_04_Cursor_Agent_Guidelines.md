# Skill 04：Cursor Agent 代码改写规范

**Tags:** `#cursor #agent #refactor #safe-edit`

## L0 决策卡（30 秒）

- 修改 >20 行且结构变动大？优先整段重写，避免碎片补丁
- 涉及多层引号/正则？避免 inline 命令
- 后台脚本需要人工介入？禁止阻塞 `input()`

## L1 标准流程（最小执行）

### 1) 编辑策略

- 小改：精准替换（必须唯一上下文）
- 大改：整体重写文件，避免缩进污染

### 2) 自动化脚本规范

- 禁用阻塞输入
- 人工验证场景用轮询等待

```python
while "验证码" in page_text:
    print("等待人工处理验证码...")
    time.sleep(5)
```

### 3) 变更后验证

- 先跑最小命令验证
- 再跑全流程

## L2 故障处理

- `IndentationError`：优先怀疑补丁残留，改为全量重写
- 超时：检查是否有隐含阻塞（input/死等待）
- 新旧逻辑混杂：回退后按模块分段重写

## 最小输入字段

- 目标文件路径
- 期望行为（1 句话）
- 验证命令
