# Skill 98：分级调用策略（Invoke Policy）

**Tags:** `#invoke #min-context #policy #routing`

## 目标

把 Skill 的“分级结构”变成“分级调用行为”，避免一次性加载全文。

## 强制调用规则（默认）

1. 先读 `Skill_00_Router_MinContext.md` 的 L0 分流。
2. 只加载 **1 个主 Skill** 的 `L0 + L1`。
3. 最多附加 **1 个辅助 Skill** 的 `L1` 片段。
4. **禁止加载 L2**，除非用户明确说：`进入L2排障`。
5. 输出限制为：`3步动作 + 1条成功判定 + 1条回滚方案`。

## 升级条件（何时允许进入 L2）

仅当满足其一：

- 出现明确报错（含错误信息）
- L1 执行后未达成功判定
- 用户明确要求深排障

## 调用模板（复制即用）

```text
只按分级调用：
1) 先读 Skill_00 的 L0 分流
2) 只加载目标 Skill 的 L0 + L1
3) 禁止加载任何 L2，除非我明确说“进入L2排障”
4) 输出限制：3步动作 + 1条成功判定 + 1条回滚方案
```

## 冲突处理

- 若多个 Skill 都像相关：优先“离当前报错最近”的那个
- 若需要跨域：主 Skill 保持不变，只短引辅助 Skill 的命令模板
- 若上下文仍过长：删案例、删背景，只保留字段表与命令
