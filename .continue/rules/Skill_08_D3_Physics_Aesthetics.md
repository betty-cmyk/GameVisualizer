# Skill 08：D3.js 物理美学与星系排版法则 (Topology & Galaxy Aesthetics)
**Tags:** `#d3js #canvas #physics #visualization`

## 痛点与血泪教训
不要试图用纯数学公式（如 `Math.cos/sin` 强算每个节点的绝对坐标）来制造星云形态！这样会抹杀真实的学术引用连线带来的“藤蔓感”，导致节点拥挤成僵硬的面团。

## L1 核心物理排版法则：数学骨架 + 拓扑血肉
要形成有真实学术脉络的长悬臂星云，必须结合以下三者：
1. **极强的万有斥力 (Charge)**：`strength(-150)` 甚至更高，这是把拥挤的类目撑开、炸出悬臂厚度的关键。
2. **原生的弹簧拉力 (Link)**：基于真实的边数据赋予弹性（如 `strength(0.04)`），让具有传承关系的论文互相拉扯，形成树枝状串联。
3. **对数螺旋跑道 (Spiral Guide)**：在 `forceX/Y` 中，用极长的弧度跑道（如 `1.8 ~ 2.5 rad`）和宽松的半径差（如 `±100px`）作为向心引导。**只做引导，不写死坐标**。

## L2 视觉与交互教训
1. **打破正圆分布**：大分类的向心点切忌完美的等距圆，必须使用手工波动数组（如 `[1.0, 1.25, 0.85...]`）制造出高低错落的自然星团感。
2. **智能防遮挡 (Smart Label Avoidance)**：Canvas 绘制文字绝对不能随视野无脑放大。必须基于 `transform.k` 动态缩小字号，并使用 8 方向螺旋扩大法 + `intersectRect` 避免标签重叠。
3. **电影级缩放 (Cinematic Zoom)**：初始选中特写时，不要把整个星系包围进去，只需精准选取锚点及其**最近的 3 个邻居**，加上少许留白，产生极强的定格特写效果。
